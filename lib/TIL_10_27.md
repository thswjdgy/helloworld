# 중간고사 복습

# 1. 플러터 소개

## 플러터가 등장한 이유는?	여러 플랫폼에서 동일한 코드로 앱 개발을 위해	iOS/Android 동시에 개발 가능 (크로스 플랫폼)
## 플러터는 어떤 언어로 개발되는가?	Dart 언어	
## 플러터의 주요 특징은?	위젯 기반 구조, 크로스 플랫폼 지원, 네이티브 성능	
## 플러터의 ‘핫 리로드(Hot Reload)’ 기능에 대한 설명으로 옳은 것은?	앱 실행 중 코드 수정 시 즉시 반영되어 빠른 테스트 가능

# 2. 다트 이해하기

## final과 const의 차이 final은 런타임에, const는 컴파일타임에 값 결정한다
## 다트의 기본 자료형이 아닌 것은?	character (Dart에는 char 없음)	
## late 키워드의 역할	변수 선언만 먼저 하고 나중에 초기화	
## ? 연산자	null 허용(nullable) 타입을 표시	
## 변수를 한 번만 할당하고 변경할 수 없게 만드는 키워드	final	
## null 값을 허용하는 변수 타입을 나타내는 기호는 	?이다	
## var로 선언한 변수는?	한 번 타입이 정해지면 변경 불가 (타입 추론)	
## 반복문이 아닌 것은?	switch문, if문 	
## OOP의 핵심 개념이 아닌 것은?	컴파일링(Compilation) (OOP 핵심은 상속, 캡슐화, 다형성, 추상화)

# 3. 플러터 필수 개념 이해

## 플러터 앱의 가장 기본 단위는?	위젯(Widget)

## Row 위젯의 정렬 축은?	가로 방향(horizontal)

# 4. 플러터의 주요 장점

## 1️⃣ 단일 코드베이스 (One Codebase for All)	한 번의 코드 작성으로 iOS, Android, 웹, 데스크톱 앱까지 모두 개발 가능. (크로스 플랫폼)
## 2️⃣ 빠른 개발 (Fast Development)	Hot Reload / Hot Restart 기능을 통해 앱을 재시작하지 않고 즉시 코드 수정 반영 가능 → UI/기능 실험이 빠름.
## 3️⃣ 높은 성능 (Near-Native Performance)	플러터는 네이티브 코드로 직접 컴파일(Dart AOT) 되므로 느린 브릿지가 없음. (React Native보다 빠름)
## 4️⃣ 일관된 UI / 디자인 (Consistent UI)	자체 렌더링 엔진(Skia)을 사용해 모든 기기에서 동일한 UI 표현 가능.
## 5️⃣ 다양한 위젯 제공 (Rich Widgets)	Material, Cupertino 등 다양한 스타일의 위젯 내장, 커스터마이징도 쉬움.
## 6️⃣ 오픈소스 / 커뮤니티 활발 (Open Source & Active Community)	구글이 관리하는 오픈소스 프로젝트로, 플러그인/패키지 생태계가 매우 풍부함.
## 7️⃣ 생산성 향상 (High Productivity)	코드량이 적고 구조가 단순하여 유지보수 및 협업 효율이 높음.
